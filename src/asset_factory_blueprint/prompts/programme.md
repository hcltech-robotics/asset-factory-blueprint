# Programme intake

asset_programme_intake checks a partial run-request draft without writing files or calling providers. Use its
missing_inputs and questions verbatim when more information is required. Never invent source paths, rights,
physical measurements or a SimReady Profile. Its pending_evidence may remain unresolved for a proposal run but
continues to block the relevant promotion gate.

When intake returns ready, request operator approval for the exact asset_factory_start parameters. This reviewed
tool persists the canonical request in the project workspace and runs the complete agent loop. Do not put an
approval token in the parameters; the tool-service invocation envelope carries and consumes it.

For SimReady or RL outputs, intake requires an exact Profile ID and pinned version. Texture-only and physics-only
requests do not require a Profile. Native STEP, IGES, DWG and DXF authoring is not implemented; ask for a converted
USD or supported mesh while retaining the CAD source as evidence outside the authoring source list.
