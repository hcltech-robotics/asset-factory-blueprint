# Support

## Where to ask

- Use GitHub issues for reproducible defects, documentation gaps and supported-environment questions.
- Use GitHub discussions, when enabled, for design and integration questions that are not defects.
- Report vulnerabilities privately as described in [SECURITY.md](SECURITY.md).
- Report conduct concerns privately as described in [CODE_OF_CONDUCT.md](CODE_OF_CONDUCT.md).

## Diagnostic bundle

Include the software version, operating system, Python version, relevant runtime versions, command line with secrets removed, the smallest run request that reproduces the problem and the resulting report or traceback. For GPU or Isaac Sim faults, include the GPU model, driver, CUDA, OpenUSD and Isaac Sim versions.

Do not attach provider keys, bearer tokens, signed URLs, proprietary source assets or unrestricted project workspaces. Replace sensitive paths and source material with a minimal fixture.

## Support boundary

The compatibility table in [docs/support-matrix.md](docs/support-matrix.md) defines verified and provisional targets. Maintainers may still help with unlisted targets, but an unlisted combination is not a compatibility promise. External reconstruction backends, model weights and hosted providers retain their own support and licence boundaries.
